## Projects

### 1. Sales Dashboard вЂ” `projects/sales-dashboard/`

Interactive sales dashboard built with **Streamlit + Plotly**. Deployed on **Streamlit Cloud** (free tier), linked from the portfolio page.

**Stack:** Python 3.11, Streamlit, Plotly, Pandas, NumPy

**Dataset:** Public Kaggle sales dataset (CSV committed to repo under `data/`)

**Key metrics & charts:**
- KPI cards: total revenue, average ticket, total orders, goal attainment %
- Monthly revenue вЂ” line chart with year-over-year comparison
- Revenue by category вЂ” sorted horizontal bar chart
- Regional heatmap вЂ” choropleth of Brazil (regions)
- Top sellers вЂ” bar chart with delta vs previous period
- Value distribution вЂ” histogram / box plot by category

**Sidebar filters:** date range, year, category, region, seller

**Files:**

HR analytics report built with **Looker Studio**, connected live to a **Google Sheets** datasource.

- Turnover rate by department вЂ” bar chart
- Absenteeism heatmap вЂ” calendar or matrix view

### 4. Python Data Pipeline вЂ” `python/data-pipeline/`

Standalone Python ETL project demonstrating data engineering skills. Collects data from a public API, transforms it with Pandas, and exports a clean report to Excel.

**Stack:** Python 3.11, Pandas, Requests, openpyxl, schedule (optional for automation)

**Source:** Any free public API (e.g. Brasil API, Open Exchange Rates, or a public dataset endpoint)

**Pipeline steps:**
1. `extract.py` вЂ” fetches raw data from the API
2. `transform.py` вЂ” cleans, normalizes, and enriches the data
3. `load.py` вЂ” exports final DataFrame to a formatted `.xlsx` report
4. `pipeline.py` вЂ” orchestrates the three steps end-to-end

**Files:**
```
data-pipeline/
в”њв”Ђв”Ђ pipeline.py             # Main entrypoint: runs extract в†’ transform в†’ load
в”њв”Ђв”Ђ extract.py
в”њв”Ђв”Ђ transform.py
в”њв”Ђв”Ђ load.py
в”њв”Ђв”Ђ output/
в”‚   в””в”Ђв”Ђ report.xlsx         # Generated report (gitignored)
в”њв”Ђв”Ђ tests/
в”‚   в”њв”Ђв”Ђ test_transform.py
в”‚   в””в”Ђв”Ђ test_load.py
в”њв”Ђв”Ђ requirements.txt
в””в”Ђв”Ђ README.md
```

**Conventions:**
- Each step (`extract`, `transform`, `load`) is a pure function that takes a DataFrame and returns a DataFrame вЂ” no side effects between steps
- Secrets (API keys) always in `.env`, loaded via `python-dotenv` вЂ” never hardcoded
- `output/` is gitignored; only the script that generates it is committed
- Tests cover at least the transform step (column existence, null checks, type validation)
