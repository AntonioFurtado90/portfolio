## Projects

### 1. Sales Dashboard — `projects/sales-dashboard/`

Interactive sales dashboard built with **Streamlit + Plotly**. Deployed on **Streamlit Cloud** (free tier), linked from the portfolio page.

**Stack:** Python 3.11, Streamlit, Plotly, Pandas, NumPy

**Dataset:** Public Kaggle sales dataset (CSV committed to repo under `data/`)

**Key metrics & charts:**
- KPI cards: total revenue, average ticket, total orders, goal attainment %
- Monthly revenue — line chart with year-over-year comparison
- Revenue by category — sorted horizontal bar chart
- Regional heatmap — choropleth of Brazil (regions)
- Top sellers — bar chart with delta vs previous period
- Value distribution — histogram / box plot by category

**Sidebar filters:** date range, year, category, region, seller

**Files:**
```
sales-dashboard/
├── app.py                  # Streamlit entrypoint
├── data/
│   └── sales.csv
├── components/
│   ├── kpis.py
│   ├── charts.py
│   └── filters.py
├── utils/
│   ├── data_loader.py
│   └── formatters.py
├── styles/
│   └── theme.py            # Color palette and global layout constants
├── requirements.txt
└── README.md
```

**Color palette (defined in `styles/theme.py`):**
```python
ACCENT      = "#C8F04A"   # lime green — primary highlight
ACCENT2     = "#7B61FF"   # purple — secondary highlight
ACCENT3     = "#FF6B6B"   # red — drop / alert
ACCENT4     = "#FFB84C"   # amber — neutral / warning
BG_MAIN     = "#0D0D0F"
BG_CARD     = "#16161A"
GRID_COLOR  = "#1E1E24"
TEXT_MAIN   = "#E8E6E0"
TEXT_MUTED  = "#6B6B7A"
```

All Plotly charts use `paper_bgcolor="rgba(0,0,0,0)"` and `plot_bgcolor="rgba(0,0,0,0)"` to inherit the app background.

**Conventions:**
- Chart functions always return `go.Figure` — never call `st.plotly_chart()` inside them
- `@st.cache_data` on every function that loads or transforms heavy data
- Business-domain variable names in Portuguese (`receita_total`), function names in English (`get_revenue_by_month`)
- All color and layout constants in `styles/theme.py` — never hardcoded in components
- Sidebar logic always in `components/filters.py`, never inline in `app.py`

**Deploy:** Streamlit Cloud — push to `main` triggers auto-deploy.

### 2. HR Analysis — `projects/hr-analysis/`

HR analytics report built with **Looker Studio**, connected live to a **Google Sheets** datasource.

**Stack:** Looker Studio, Google Sheets (as database), optional Python script to populate the sheet

**Dataset:** Synthetic HR dataset generated by a Python script and pushed to Google Sheets via the Sheets API.

**Key metrics & charts:**
- Headcount over time — line chart
- Turnover rate by department — bar chart
- Absenteeism heatmap — calendar or matrix view
- Tenure distribution — histogram
- Active vs terminated employees — donut chart
- KPI scorecard: total headcount, monthly turnover %, avg tenure

**Files:**
```
hr-analysis/
├── generate_data.py        # Generates synthetic HR data and pushes to Google Sheets
├── requirements.txt        # google-auth, gspread, pandas, faker
├── looker_studio_link.md   # Public report URL + embed instructions
├── screenshots/
└── README.md
```

**Conventions:**
- The Google Sheet acts as the single source of truth — never commit raw data CSVs here
- `generate_data.py` must be idempotent: running it twice should not duplicate rows (use a unique employee ID as key)
- Looker Studio report must be set to "Anyone with the link can view"

### 3. Financial Panel — `projects/financial-panel/`

Simplified P&L, cash flow, and budget vs actual dashboard. Primary version in **Power BI**, with an alternative in **Apache Superset** for clients who prefer open-source.

**Stack (primary):** Power BI Desktop, Power Query, DAX
**Stack (alternative):** Apache Superset, SQLite or DuckDB, SQL

**Dataset:** Synthetic financial data (monthly P&L entries) — CSV committed to repo.

**Key metrics & charts:**
- Simplified P&L (DRE): gross revenue, COGS, gross margin, EBITDA
- Cash flow waterfall chart — inflows vs outflows
- Budget vs actual — grouped bar with variance line
- Monthly trend — area chart for revenue and expenses
- KPI cards: net revenue, gross margin %, burn rate, months of runway

**Files:**
```
financial-panel/
├── data/
│   └── financials.csv      # Monthly entries: date, account, category, actual, budget
├── generate_data.py        # Generates synthetic financials dataset
├── financial_panel.pbix    # Power BI file
├── superset/
│   ├── dashboard_export/   # Superset dashboard JSON export
│   └── queries.sql         # Chart SQL queries documented
├── screenshots/
└── README.md
```

**DAX conventions (Power BI):**
- All measures in a dedicated `_Measures` table, never inside dimension tables
- Measure names follow pattern: `[Revenue YTD]`, `[Gross Margin %]`
- Use `CALCULATE` + `DATEYTD` / `SAMEPERIODLASTYEAR` for time intelligence

### 4. Python Data Pipeline — `python/data-pipeline/`

Standalone Python ETL project demonstrating data engineering skills. Collects data from a public API, transforms it with Pandas, and exports a clean report to Excel.

**Stack:** Python 3.11, Pandas, Requests, openpyxl, schedule (optional for automation)

**Source:** Any free public API (e.g. Brasil API, Open Exchange Rates, or a public dataset endpoint)

**Pipeline steps:**
1. `extract.py` — fetches raw data from the API
2. `transform.py` — cleans, normalizes, and enriches the data
3. `load.py` — exports final DataFrame to a formatted `.xlsx` report
4. `pipeline.py` — orchestrates the three steps end-to-end

**Files:**
```
data-pipeline/
├── pipeline.py             # Main entrypoint: runs extract → transform → load
├── extract.py
├── transform.py
├── load.py
├── output/
│   └── report.xlsx         # Generated report (gitignored)
├── tests/
│   ├── test_transform.py
│   └── test_load.py
├── requirements.txt
└── README.md
```

**Conventions:**
- Each step (`extract`, `transform`, `load`) is a pure function that takes a DataFrame and returns a DataFrame — no side effects between steps
- Secrets (API keys) always in `.env`, loaded via `python-dotenv` — never hardcoded
- `output/` is gitignored; only the script that generates it is committed
- Tests cover at least the transform step (column existence, null checks, type validation)

## GitHub Pages (portfolio website)

The root `index.html` is the public-facing portfolio page, auto-deployed via GitHub Pages from the `main` branch.

**Setup:**
1. Go to repository **Settings → Pages**
2. Set source to **Deploy from branch → main → / (root)**

**Each project card on the page links to:**
- Live demo (Streamlit Cloud URL, Looker Studio public link, or Superset embed)
